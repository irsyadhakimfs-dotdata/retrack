> **Dokumen ini adalah hasil elaborasi senior dari spesifikasi kasar yang Anda berikan. Tujuannya: mengubah "wishlist fitur" menjadi spesifikasi yang siap dieksekusi oleh AI coding assistant (Claude Code) tanpa ambiguitas.**

---

## 1. Ringkasan Produk

**ReTrack** adalah aplikasi web personal finance tracker berbasis Flask (Python) dengan fokus pada:

- Pencatatan transaksi (pemasukan & pengeluaran) multi-wallet
- Visualisasi finansial interaktif (dashboard, charts)
- Budgeting & goal-tracking (Plans)
- OCR struk belanja via Gemini AI (input pengeluaran via foto)
- Market Overview kurs USD/IDR untuk konteks daya beli rupiah

Target user: **personal use** (single-tenant atau multi-tenant dengan login per-user).

## 2. Stack Teknologi (Keputusan Final)

| Layer | Pilihan | Alasan |
|---|---|---|
| **Backend** | Flask 3.x + Python 3.11+ | Sudah ditentukan user; minimal & fleksibel |
| **ORM** | SQLAlchemy 2.x + Flask-SQLAlchemy | Standard de facto, future-proof |
| **Migration** | Flask-Migrate (Alembic) | Schema versioning, aman untuk perubahan struktur |
| **Database** | SQLite (dev/MVP) → siap migrasi ke PostgreSQL | Zero-config, file-based, cocok untuk personal app |
| **Auth** | Flask-Login + Flask-Bcrypt | Battle-tested, simpel |
| **Forms** | Flask-WTF + WTForms | CSRF protection built-in |
| **Template Engine** | Jinja2 (bawaan Flask) | Server-rendered, SEO-friendly, simpel |
| **CSS Framework** | Tailwind CSS 3.x (via CDN dulu, lalu build) | Utility-first, glassmorphism-friendly |
| **JS Reactivity** | Alpine.js 3.x | 15kb, deklaratif, tanpa build step |
| **Charting** | Chart.js 4.x | Ringan, interaktif, support line/bar/pie/mixed |
| **AI OCR** | Google Gemini API (gemini-2.0-flash atau gemini-2.5-flash) | Multimodal, gratis tier, mudah integrasi |
| **Forex Data** | `yfinance` (Yahoo Finance) | Historical 5 tahun, gratis, no API key |
| **File Upload** | Flask built-in + Pillow untuk validasi gambar | Standar |
| **Export** | `pandas` + `openpyxl` | CSV & XLSX export |
| **Env Management** | `python-dotenv` | Pisahkan secret dari kode |

### Kenapa Jinja2 + Alpine.js (bukan React)?

1. **Glassmorphism murni CSS** — Tidak butuh framework JS untuk efek visual (`backdrop-filter: blur(20px)`, gradien, transparansi).
2. **Tidak ada build pipeline** — Tidak perlu Vite/Webpack/Node setup, langsung jalan dengan `flask run`.
3. **Alpine.js cukup untuk interaktivitas** — Toggle dark mode, modal form, dropdown, tab switcher, chart re-render: semua bisa dengan `x-data`, `x-show`, `x-on:click`.
4. **Belajar Flask end-to-end** — Anda mahasiswa Data Science yang ingin belajar mobile/web app; pisah backend-frontend dengan REST API justru menambah kompleksitas yang belum perlu.
5. **Kurva belajar landai** — Mahasiswa Data Science biasanya familiar dengan templating (Jupyter, dll); Jinja2 mirip secara konsep.

## 4. Database Schema (ERD Konseptual)

### Tabel-tabel inti

#### `users`
- `id` (PK)
- `username` (unique)
- `email` (unique)
- `password_hash`
- `created_at`
- `theme_preference` ('light' | 'dark')

#### `wallets`
- `id` (PK)
- `user_id` (FK → users)
- `name` (e.g. "BCA", "GoPay", "Cash")
- `wallet_type` ('bank' | 'ewallet' | 'cash' | 'other')
- `icon` (Google Material Icon name, e.g. "account_balance")
- `initial_balance` (Decimal)
- `created_at`

> Saldo current = `initial_balance + SUM(incomes) - SUM(expenses)` (dihitung, bukan disimpan)

#### `categories`
- `id` (PK)
- `user_id` (FK)
- `name`
- `kind` ('income' | 'expense')
- `icon` (Google Material Icon name)
- `color` (hex)
- `parent_id` (FK self, nullable → untuk subcategory)

#### `transactions`
- `id` (PK)
- `user_id` (FK)
- `wallet_id` (FK → wallets)
- `category_id` (FK → categories, nullable)
- `type` ('income' | 'expense')
- `amount` (Decimal, positive)
- `transaction_date` (Date)
- `transaction_time` (Time, default 00:00)
- `description` (Text, nullable)
- `source` ('manual' | 'ocr' | 'import')
- `receipt_image_path` (nullable, untuk OCR)
- `created_at`, `updated_at`

#### `budgets`
- `id` (PK)
- `user_id` (FK)
- `category_id` (FK, nullable — bisa per-kategori atau global)
- `amount_limit` (Decimal)
- `period` ('monthly' | 'weekly' | 'yearly')
- `start_date`, `end_date`
- `is_active`

#### `saving_plans`
- `id` (PK)
- `user_id` (FK)
- `name` (e.g. "Beli laptop")
- `target_amount` (Decimal)
- `current_amount` (Decimal, accumulated)
- `target_date` (nullable)
- `icon`, `color`
- `is_completed`

#### `exchange_rates_cache`
- `id` (PK)
- `date`
- `currency_pair` (e.g. 'USDIDR=X')
- `rate` (Decimal)
- `fetched_at`
- UNIQUE(date, currency_pair)

2. **3 Stat Cards**: Total Saldo, Total Pengeluaran (filtered), Total Pemasukan (filtered)

### 5.5 Wallets (`/wallets`)
- Grid card per wallet (glassmorphism) dengan:
  - Nama, ikon, saldo current
  - Tombol "Tambah Pemasukan" cepat
  - Tombol edit/delete (3-dot menu)
- Modal "Tambah Wallet": nama, tipe, ikon (picker Google Icons), saldo awal
- Modal "Tambah Pemasukan ke Wallet": amount, kategori, tanggal, waktu, deskripsi

### 5.6 Financial Report (`/financial-report`)
Halaman terpenting. Komponen:

1. **Filter bar**: tanggal range, wallet, kategori, tipe (income/expense), search by description
2. **Action buttons**:
   - "+ Tambah Pemasukan" (modal)
   - "+ Tambah Pengeluaran" (modal)
   - "📷 Scan Struk" (upload foto → Gemini OCR)
   - "Export CSV" / "Export XLSX"
   - "Kelola Kategori" → modal/redirect ke Plans
3. **Tabel transaksi** (paginated, default 20 per halaman):
   - Tanggal, Waktu, Tipe, Kategori, Wallet, Jumlah, Deskripsi, Actions (edit/delete)
   - Sortable per kolom
4. **Modal CRUD** untuk income/expense dengan form Flask-WTF
5. **OCR Flow**:
   - User upload foto struk
   - Backend kirim ke Gemini API dengan prompt terstruktur
   - Gemini return JSON: `{merchant, total, date, time, items: [{name, price, category_suggestion}]}`
   - Backend pre-fill form modal, user review & confirm
   - Jika data tidak lengkap, kosongkan field dan user isi manual

### 5.7 Plans (`/plans`)
Dua sub-section dalam satu halaman (atau tab):

**A. Categories Management**
- Grid card per kategori dengan ikon Google Material Icons
- CRUD kategori (nama, ikon, warna, parent untuk subcategory)
- Filter: income vs expense kategori

**B. Budget Settings**
- Form: "Set anggaran maksimal untuk kategori X = Rp Y per bulan"
- List budget aktif dengan progress
- Bisa override per subcategory

**C. Saving Plans / Goals**
- Card per goal: nama, target, current, target date, % progress
- Tombol "Tambah Dana ke Goal" (mengurangi wallet, menambah current_amount goal)

### 5.8 Market Overview (`/market-overview`)
- Line chart kurs USD/IDR dengan toggle range: 7D, 1M, 1Y, 5Y
- Plot titik (scatter overlay) untuk setiap pemasukan user
- Hover tooltip per titik:
  - Tanggal pemasukan
  - Nominal pemasukan (Rp)
  - Kurs saat itu
  - Kurs sekarang
  - **% perubahan daya beli** dengan rumus:
    ```
    ((kurs_sekarang - kurs_tanggal_X) / kurs_tanggal_X) * 100%
    ```
    Jika positif → rupiah melemah → daya beli berkurang
- Penjelasan singkat di bawah chart tentang interpretasi data

## 7. Integrasi Eksternal

### 7.1 Gemini API (OCR Struk)
- Model: `gemini-2.5-flash` (cepat, multimodal, cukup untuk OCR struk)
- API Key disimpan di `.env`: `GEMINI_API_KEY=...`
- Library: `google-generativeai` (pip)
- Prompt template harus minta JSON output:
  ```
  Analyze this receipt image. Return STRICT JSON with:
  {
    "merchant": "...",
    "total_amount": <number>,
    "date": "YYYY-MM-DD",
    "time": "HH:MM",
    "items": [{"name": "...", "amount": <number>}],
    "suggested_category": "Food|Transport|..."
  }
  If any field is unreadable, set it to null.
  ```
- Validation di backend: parse JSON, validate types, fallback ke null jika gagal
- User review hasil sebelum save (jangan auto-commit)

### 7.2 yfinance (Forex)
- Library: `yfinance`
- Symbol: `IDR=X` (USD to IDR) atau `USDIDR=X`
- Cache di tabel `exchange_rates_cache` agar tidak hit API setiap request
- Refresh strategy: cek timestamp terakhir, fetch jika > 12 jam
- Untuk historical 5 tahun: `ticker.history(period='5y')`

## 8. Keamanan & Best Practices

1. **Secrets** — Semua API key, SECRET_KEY di `.env`, jangan di-commit. Sediakan `.env.example`.
2. **Password hashing** — Bcrypt (cost factor 12)
3. **CSRF** — Aktifkan Flask-WTF CSRF di semua form
4. **SQL Injection** — Selalu pakai ORM, jangan raw query string concat
5. **File Upload** — Validasi ekstensi (jpg/png/webp only), max 5MB, scan dengan Pillow untuk pastikan valid image
6. **Rate Limiting** — Flask-Limiter untuk endpoint OCR (mahal di Gemini)
7. **User Isolation** — SEMUA query harus filter `WHERE user_id = current_user.id`
8. **Decimal untuk uang** — JANGAN pakai float! Pakai `Numeric(15, 2)` di SQLAlchemy

## 11. Catatan Khusus untuk Vibe Coding

1. **JANGAN minta Claude Code build semuanya sekaligus.** Lakukan per phase, per blueprint.
2. **Selalu verify migrasi** sebelum lanjut: `flask db migrate -m "..."` → review file → `flask db upgrade`
3. **Test setiap endpoint** dengan browser sebelum lanjut ke fitur berikutnya
4. **Commit per fitur** ke git — kalau Claude Code rusak sesuatu, mudah revert
5. **Simpan `.env` SEKARANG** di luar repo, dan tambahkan ke `.gitignore` SEBELUM commit pertama
6. **GEMINI_API_KEY yang Anda share di konteks — ROTATE/regenerate setelah deploy publik** karena sudah ter-expose
